# Описание решения — Кредитный скоринг (прогноз дефолта)

## 1. Постановка задачи

Задача — предсказать вероятность выхода клиента в дефолт по кредиту (неплатёж более 3 месяцев в течение года) по истории его кредитных продуктов. Данные в long-формате: на каждого клиента (`id`) приходится от 1 до 55 строк — по одной на каждый кредитный продукт в его истории (упорядочены по `rn`). Метка дефолта (`flag`) — одна на клиента. Метрика — ROC-AUC. Дисбаланс классов: ~3.55% дефолтов. Train/test разделены по времени (более поздние `id` — тест).

Итоговый результат: **OOF ROC-AUC = 0.78391 → public LB = 0.7836.**

Разведочный анализ данных — в ноутбуке [`notebooks/eda_final.ipynb`](notebooks/eda_final.ipynb) (дисбаланс, длины последовательностей, кардинальности кодов, одномерная сила признаков, train/test дрейф).

## 2. Архитектура решения

Решение — **rank-бленд пяти баз поверх двух представлений данных**. Главный рычаг — sequence-ветка с per-column эмбеддингами; GBDT-агрегаты вышли на плато и в финале весят ≈0, оставаясь как дешёвая страховка.

```
                          ┌─────────────────────────────────┐
                          │    Исходные данные (long)       │
                          │  18.3M строк / 2.1M клиентов    │
                          │  59 признаков × rn продуктов    │
                          └──────────┬───────────┬──────────┘
                                     │           │
                    ┌────────────────┘           └────────────────┐
                    ▼                                             ▼
          АГРЕГАТЫ НА ID                              ПОСЛЕДОВАТЕЛЬНОСТИ (59 кодов/шаг)
          182 признака                                 per-column embeddings
              │                                     ┌────────┼─────────┐
        ┌─────┴─────┐                               ▼        ▼         ▼
        ▼           ▼                          bi-GRU ×3   attn-GRU  Transformer ×3
     LightGBM   CatBoost                       сиды 42/101/202   (pool)   сиды 42/101/202
     0.75478    0.75475                         усреднены   0.77878   усреднены
        │           │                            = 0.78248      │      = 0.77955
        │           │                                │          │         │
        └─────┬─────┘                                └────┬─────┴─────────┘
              │                                           │
              └───────────────────┬───────────────────────┘
                                  ▼
                  RANK-BLEND (Nelder-Mead по OOF)
       avg_bigru 0.535 + avg_transf 0.318 + attn 0.120 + lgbm 0.027 + cat 0.0
                  ─────────────────────────────────────
                       OOF ROC-AUC = 0.78391  →  LB 0.7836
```

### 2.1. Ветка агрегатов → GBDT (плато, вес ≈0)

История клиента сворачивается в один вектор фиксированной длины (mean/max/min/std/counts по 8 блокам признаков) → **182 признака**. LightGBM (`scale_pos_weight≈27`) и CatBoost (`auto_class_weights=Balanced`) на 5 фолдах. Оба упираются в **OOF 0.7548**. Ablation feature sets (`v2`, `v3`) показал, что производные агрегаты сигнала не добавляют — весь дополнительный сигнал в **порядке** продуктов, который агрегаты уничтожают.

### 2.2. Ветка последовательностей → эмбеддинговые seq-модели (рычаг)

Все 59 признаков — **категориальные коды** (бины 0–19, статусы, флаги), поэтому каждый получает собственный обучаемый эмбеддинг (общая таблица `nn.Embedding` + per-column offsets, словарь = `max(train,test)+1`). Эмбеддинги конкатенируются по таймстепу. Три архитектуры над этим общим фронт-эндом:

- **bi-GRU** (hidden=128, emb=8, max_len=40, last-hidden pooling). Одиночный seed-42 OOF = **0.77982** — скачок **+0.025** к scaled-numeric GRU (0.7547) исключительно за счёт смены представления входа. Это же неявно решило проблему sentinel-кодов: коды эмбеддятся, числового пути для «отравления» нет.
- **attention-pooling bi-GRU** (hidden=192, emb=12, max_len=55). OOF = **0.77878**. Диверсификатор.
- **Transformer-энкодер** (d_model=128, 2 слоя, 4 головы, learned positions, pre-LN). OOF = **0.77447** — слабее в одиночку, но другой индуктивный bias.

Обучение на MPS (Apple), `BCEWithLogitsLoss` с `pos_weight`, паддинг маскируется, early stopping по fold-validation (patience=2).

### 2.3. Мульти-сид усреднение (+0.0019 bi-GRU, +0.0051 transformer)

bi-GRU и transformer обучены на сидах 42/101/202 при **фиксированных фолдах** (seed=42 в `make_folds`, варьируется только init модели через `--model-seed`). Усреднение вероятностей трёх сидов:

- **bi-GRU:** OOF **0.78248** против 0.78056 у лучшего одиночного — **+0.00192**. Усреднённый bi-GRU в одиночку уже бьёт прежний 4-моделный бленд.
- **Transformer:** OOF **0.77955** против 0.77516 у лучшего одиночного — **+0.00440**. Эффект мульти-сида ещё больше, чем у bi-GRU (более высокая дисперсия инициализации у self-attention).

### 2.4. Ансамбль (rank-blend)

Предсказания каждой базы ранг-нормализуются (0..1) — выравнивает масштабы GBDT-вероятностей и NN-сигмоиды; ROC-AUC ранговая, поэтому корректно. Веса — Nelder-Mead по OOF ROC-AUC (неотрицательные, сумма=1). OOF/test выравниваются по **значению `id`**.

Финальные веса: **avg_bigru 0.535 + avg_transformer 0.318 + attn 0.120 + lgbm 0.027 + cat 0.0**, OOF = **0.78391**. Мульти-сид transformer усилился с веса 0.21 до 0.32 — мульти-сид снизил дисперсию и повысил полезность декоррелированной базы.

## 3. Ключевые решения и обоснования

### Почему эмбеддинги, а не сырые числа?
Все 59 признаков — целочисленные коды. Поданные как числа, они навязывают модели ложный линейный порядок для категориальных кодов (`credit_type=2` не «больше» 1). Эмбеддинги дают каждому коду обучаемое представление. Эффект: +0.025 OOF к scaled-numeric GRU.

### Почему мульти-сид усиливает результат?
Одиночная NN имеет fold-std ≈0.0024 (дисперсия инициализации/обучения). Усреднение сидов на одних фолдах снижает эту дисперсию без утечки — каждый OOF валиден на своём разбиении. Дало +0.0019, надёжнее любого твика блендера.

### Почему слабый transformer помогает ансамблю?
Корреляция transformer↔avg_bigru = **0.943**, заметно ниже, чем attn↔bigru (0.962). Несмотря на OOF 0.77447 (слабейший), его декорреляция (другой индуктивный bias — self-attention vs рекуррентность) даёт блендеру новый сигнал → вес 0.21 и **+0.0004** к финалу.

### Почему rank-blend, а не stacking/AutoGluon?
Проверено: AutoGluon-мета поверх OOF-предсказаний **в точности повторяет** линейный rank-blend (0.78201), а обогащение 182 агрегатами **вредит** (≈−0.0005 на парном 800k-тесте) — GBDT-OOF уже дистиллировали агрегаты. Вывод: бутылочное горлышко не в блендере, а в базовых моделях. Линейного rank-blend достаточно.

### Почему StratifiedKFold, а не TimeSeriesSplit?
После агрегации/группировки на каждый `id` одна метка → `StratifiedKFold(5, seed=42)` по `id` корректен (нет утечки строк, стратификация важна при дисбалансе). Опасение по временному дрейфу **снято эмпирически** (см. ниже).

## 4. Результаты

### OOF ROC-AUC (5-fold, id-level)

| модель | OOF | примечание |
|---|---|---|
| LightGBM / CatBoost | 0.7548 | плато агрегатов |
| Embedding bi-GRU (seed 42) | 0.77982 | +0.025 к scaled-numeric |
| bi-GRU сиды 101 / 202 | 0.78056 / 0.78047 | |
| **avg bi-GRU (3 сида)** | **0.78248** | +0.0019 мульти-сид |
| attention bi-GRU | 0.77878 | диверсификатор |
| Transformer encoder (seed 42) | 0.77447 | декоррелирован (0.943) |
| Transformer сиды 101 / 202 | 0.77516 / 0.77468 | |
| **avg Transformer (3 сида)** | **0.77955** | +0.0051 мульти-сид |

### Прогресс блендов

| бленд | OOF | Δ |
|---|---|---|
| lgbm+cat+bigru (база) | 0.78079 | — |
| + attn | 0.78201 | +0.0012 |
| swap → avg_bigru | 0.78301 | +0.0010 |
| + transformer (5-way) | 0.78340 | +0.0004 |
| **swap → avg_transformer (финал)** | **0.78391** | **+0.0005** |

### Лидерборд и валидация OOF→LB

| дата | сабмит | OOF | Public |
|---|---|---|---|
| 06-18 | LightGBM baseline | 0.7548 | 0.75591 |
| 06-18 | Ансамбль v1 (LGBM+Cat+GRU) | 0.7645 | 0.76456 |
| 06-19 | Ансамбль v2 (LGBM+Cat+EmbGRU) | 0.7808 | 0.78261 |
| 06-20 | 5-way (avg_bigru+transf+attn) | 0.78340 | 0.7835 |
| **06-20** | **5-way (avg_bigru+avg_transf+attn)** | **0.78391** | **0.7836** |

**Гэп OOF→LB ≈ 0** (подтверждено двумя подачами). Random-CV OOF — надёжный прокси лидерборда: приросты можно принимать по OOF **без траты загрузок**. На последней подаче +0.0005 OOF → +0.0001 LB — мы на **шумовом полу**: суб-0.001 улучшения OOF не конвертируются в LB. Изначальное опасение временного дрейфа (time-holdout прогнозировал −0.025) не подтвердилось.

## 5. Известные ограничения

1. **GBDT-агрегаты на плато (вес ≈0):** весь сигнал ушёл в seq-ветку; ручной feature engineering и stacking через агрегаты проверены отрицательно.
2. **Stacking через эмбеддинги не делался:** OOF-представления seq-моделей не подавались в GBDT как фичи (требует отдельного leak-safe CV-слоя).
3. **Калибровка не проводилась:** ROC-AUC ранговая, на скор не влияет.
4. **Шумовой пол LB:** последняя подача показала, что +0.0005 OOF → +0.0001 LB; дальнейшие микро-улучшения не конвертируются.

## 6. Направления улучшения (по приоритету, всё по OOF)

| # | направление | ожидаемый Δ OOF | статус |
|---|---|---|---|
| 1 | TCN — третья декоррелированная архитектура | диверсификация | план |
| 2 | Мульти-сид attn | мелкий | план |
**Закрытые направления (проверены, отрицательны):** AutoGluon как блендер (повторяет rank-blend); обогащение мета-матрицы агрегатами (вредит); GBDT feature sets v2/v3 (шум); scaled-numeric GRU (−0.025 к эмбеддингам); **мульти-сид transformer** (реализован: +0.0005 OOF / +0.0001 LB → шумовой пол); **стекинг через OOF-эмбеддинги seq → GBDT** (META OOF 0.539 — провал; 128-d penultimate-представление bi-GRU коллинеарно скалярному логиту, LightGBM не нашёл дополнительного нелинейного сигнала).

**Что НЕ делать:** калибровка (ранговая метрика), подгонка под Public (доверяем OOF), усложнение блендера (горлышко в базах).
