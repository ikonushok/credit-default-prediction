# Бриф задачи — Альфа-Банк x МФТИ «Кредитный скоринг» (прогноз дефолта)

## Цель

Предсказать вероятность выхода клиента в дефолт по кредиту (неплатёж более 3 месяцев в течение года) на основе истории его кредитных продуктов. Модель должна ранжировать клиентов по риску дефолта. Итоговый артефакт — CSV-сабмит с вероятностью дефолта на каждый `id`.

## Данные

Файлы с платформы (в `data/`):

- `data/raw/train_data.parquet` — обучающая история кредитных продуктов (без таргета);
- `data/raw/test_data.parquet` — тестовая история (без таргета);
- `data/raw/train_target.csv` — метки `id, flag` для train (`flag = 1` — дефолт);
- `data/Кредитный скорринг/sample_submission.csv` — обязательный формат сабмита (`id, flag`).

**Формат long, а не плоский.** Каждый `id` — это одна заявка; у неё несколько строк, по одной на кредитный продукт в истории клиента, упорядоченных по `rn` (1..55, в среднем ~8.7 строк на `id`). Метка в `train_target.csv` — одна на `id`. Поэтому историю нужно **агрегировать в единый вектор признаков на каждый `id`** до джойна с таргетом и обучения. Проверено: множество `id` в `train_data` == множество `id` в `train_target`; `id` в `test_data` == `id` в `sample_submission`; множества `id` train и test не пересекаются.

## Таргет и метрика

- `flag = 1` — клиент вышел в дефолт;
- `flag = 0` — клиент выплатил кредит;
- доля дефолтов ≈ **3.55%** (74 509 / 2 100 000) — сильный дисбаланс классов;
- метрика: ROC-AUC (устойчива к дисбалансу; PR-AUC считаем как вспомогательную проверку).

## Признаки (полная схема, 61 колонка, все целочисленно закодированы)

Колонки анонимизированы и предварительно закодированы в небольшие целые диапазоны (сырых денежных величин нет). Каждую колонку нужно классифицировать и использовать; текстовых или вещественных полей нет.

- идентификаторы / порядок: `id` (заявка; больший = более поздняя дата заявки), `rn` (порядок кредитного продукта внутри `id`; больший = более поздняя дата открытия);
- бинаризованные `pre_*` (бины 0–19): сроки и тайминги (`pre_since_opened`, `pre_since_confirmed`, `pre_pterm`, `pre_fterm`, `pre_till_pclose`, `pre_till_fclose`), суммы (`pre_loans_credit_limit`, `pre_loans_next_pay_summ`, `pre_loans_outstanding`, `pre_loans_total_overdue`, `pre_loans_max_overdue_sum`, `pre_loans_credit_cost_rate`), утилизация (`pre_util`, `pre_over2limit`, `pre_maxover2limit`);
- счётчики просрочек: `pre_loans5`, `pre_loans530`, `pre_loans3060`, `pre_loans6090`, `pre_loans90`;
- zero-флаги: `is_zero_loans5/530/3060/6090/90`, `is_zero_util`, `is_zero_over2limit`, `is_zero_maxover2limit`;
- последовательность платёжных статусов: `enc_paym_0..24` (помесячные статусы, коды 0–4);
- закодированные категориальные: `enc_loans_account_holder_type`, `enc_loans_credit_status`, `enc_loans_credit_type`, `enc_loans_account_cur`;
- флаги закрытия: `pclose_flag`, `fclose_flag`.

## Ограничения

Python 3.10+ и только open-source библиотеки (никаких закрытых библиотек, приватных API или заимствованного не-OSS кода). Использовать только предоставленные данные train/test/target. Сабмит — валидный CSV, точно соответствующий `sample_submission.csv` (колонки `id`, `flag`; 900 000 строк; `flag` — вещественная вероятность в `[0, 1]`). Не более 3 загрузок на платформу в день (00:00–23:59 МСК).
